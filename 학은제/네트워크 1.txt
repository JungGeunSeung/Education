1.네트워크(Network)란 통신 설비들로 두 대 이상의 컴퓨터를 서로 연결한 통신망이다.
2.인터넷(internet)은 서로 통신할 수 있는 둘 또는 그 이상의 네트워크로서, 수백만 개의 네트워크가 상호 연결된 네트워크이다.
3.다수의 컴퓨터를 네트워크로 연결했을 때 얻을 수 있는 이점은
(1) 데이터 공유 인터넷(internet)은 서로 통신할 수 있는 둘 또는 그 이상의 네트워크로서, 수백만 개의 네트워크가 상호 연결된 네트워크이다.
(2) 주변장치 공유
(3) 능률적인 통신
(4) 손쉬운 백업
4.네트워크 구성 요소는 컴퓨터 단말기, 접속장치, 전송매체이다.
5.네트워크 구성 형태는 LAN, MAN, WAN 등이 있다.

2강에서는 네트워크를 구성하는 다양한 접속장치에 대하여 학습하였습니다.
1.네트워크 접속장치는 통신망 구성에서 가장 기본이 되는 하드웨어 장치이다.
2.더미 허브는 데이터를 네트워크의 다른 컴퓨터로 전송하는 장치로, 단순히 컴퓨터와 컴퓨터 간의 네트워크를 중계하는 역할을 한다.
3.스위칭 허브는 전체 대역폭을 컴퓨터 노드 수만큼 나누어 사용하는 문제점을 해결한 접속 장치로서, 단순히 전송하는 기능을 넘어 수신지 주소로 스위칭하는 기능이 있다.
4.브리지(bridge)는 2개 이상의 근거리 통신망을 연결하여 하나의 네트워크로 만드는 장치로, 수신지 주소에 따라 특정 네트워크 트래픽만 통과시킬 수 있도록 설계된 특수한 형태의 네트워크 스위치이다.
5.라우터(router)는 서로 다른 네트워크 간에 통신하는 데 사용하는 장치로, 구조가 다른 망을 연결할 수 있다. 또한 IP 주소를 바탕으로 데이터가 수신지까지 갈 수 있는 경로를 검사하여 효율적인 경로를 선택하는 routing 기능도 수행한다.

3강에서는 네트워크의 데이터 전송방식과 망 구성형태에 대하여 학습하였습니다.
1. 교환은 갈림길에서 데이터가 전송되어야 할 경로를 선택하고, 전송은 물리적으로 1:1 연결된 시스템 사이의 데이터 전송을 한다.
2. 브로드캐스팅은 네트워크에 연결된 모든 호스트에게 데이터를 전송하는데 호스트가 많을수록 네트워크 트래픽이 증가하게 된다.
3. 네트워크 접속형태는 네트워크에 연결된 여러 노드의 물리적인 배열이 아닌 상호 연결 방법을 보여준다.
네트워크 접속 형태 - 성형, 트리형, 버스형, 링형, 그물형

4강에서는 네트워크의 전송 매체의 종류와 특징에 대하여 학습하였습니다.
1. 전송 매체는 송식 측과 수신 측을 연결하는 물리적 선로를 말하는데, 신호는 전자기적인 에너지 형태로 한 노드에서 다른 노드로 전송된다.
2. 전자기 신호는 전송 매체나 진공, 공기를 통해 이동할 수 있다.
3. 전송매체의 분류에는 유도매체인 유선 전송 매체와 비유도매체인 무선 전송매체로 나뉜다.
유선 전송 매체 - 동축 케이블, 꼬임선, 광섬유 케이블
무선 전송 매체 - 라디오파, 지상 마이크로파, 위성 마이크로파

5강에서는 네트워크에서 여러 가지 통신 방식에 대하여 학습하였습니다.
1. 네트워크에서 서버는 클라이언트(사용자)한테 요청받아 서비스를 제공하는데, 이렇게 구성된 시스템을 '클라이언트/서버 시스템'이라고 한다.
2. LAN에서 통신하는 방식에는 유니캐스트, 브로드캐스트, 멀티캐스트 방식이 있다.
3. 전송방향에 따라서 분류하면, 단방향(simplex) 통신과 양방향(duplex) 통신으로 나눌 수 있으며, 양방향 통신은 반이중(half-duplex) 통신과 전이중(full-duplex) 통신으로 나뉜다.
4. 동기화란 적절한 방법으로 송신 측에서 전송한 데이터의 각 비트를 수신 측에서 정확하게 수신할 수 있도록 속도차(클록오차)를 맞추는 것을 말한다.
5. 동기식 전송은 미리 정해진 수만큼 문자를 한 묶음으로 만들어 일시에 전송하는 방법으로, 비트와 데이터 간에 간격 없이 차례대로 비트를 전송하기 때문에 데이터는 끊어지지 않는 0과 1의 문자열로 전송된다.
6. 비동기식 전송은 긴 데이터 비트열을 연속으로 전송하는 대신 한 번에 한 문자씩 전송함으로써 수신기가 새로운 문자의 시작점에서 재동기화하도록 하는 것이다.

6강에서는 네트워크에서 오류의 발생원인과 통신 오류 검출방식에 대하여 학습하였습니다.
1. 수신 측으로 전송한 데이터는 송신 측의 데이터와 동일해야 하지만, 다양한 원인 때문에 데이터 오류가 발생할 수 있다.
2. 따라서 신뢰할 수 있는 네트워크 통신을 하려면 오류를 검출·수정해야 한다.
오류 - 단일 비트 오류, 다중 비트 오류, 잔단 오류
오류 검출 방식 - 패리티 비트 검사 방식, 블록 합 검사 방식, 순환 중복 검사 방식
3. 재전송 알고리즘(ARQ)는 통신상에서 오류가 발생했을 경우 수신측은 오류 발생을 송신측에 알리고, 송신측은 오류가 발생한 프레임을 재전송하여 오류를 수정하게 된다.
4. 대표적인 ARQ방식은 정지-대기(stop-and-wait) ARQ, Go-back-N ARQ, Selective Repeat ARQ

7강에서는 네트워크의 통신망 형태와 특징에 대하여 학습하였습니다.
1. 네트워크의 주요 통신망으로는 근거리 통신망(LAN), 대도시 통신망(MAN), 광역 통신망(WAN)이 있으며, 각 네트워크의 범주는 소유자, 규모, 거리, 물리적 구조에 따라 구분된다.
2. 데이터 충돌을 방지하려고 LAN에 연결된 모든 장치는 정의된 규칙에 따라 전송매체에 접근하는데, 이를 매체 접근 제어(MAC, Media Access Control) 방식이라고 한다.
매체 접근 방식
경쟁 방식: CSMA/CD
토큰 제어 방식: 토큰버스 방식, 토큰링 방식
3. 교환 방식에는 회선교환방식, 패킷교환방식, 메시지 교환 방식이 있다.

8강에서는 무선 네트워크의 특징, 표준, 구성, 동작에 대하여 학습하였습니다.
1.무선 LAN은 유선 LAN에 비해 고속 지원은 어렵지만, 단말기 이동의 유연함과 작업 환경을 쾌적하게 해주는 장점이 있다.
2.IEEE가 승인한 무선 LAN 표준화 기술로서, 무선 LAN 규격은 802.11a, 802.11b, 802.11g, 802.11n, 802.11ac 등이 있다.
3.802.11b 표준에서 WEP(Wired Equivalent Privacy)라는 데이터 암호화 기술을 사용하여 모든 데이터를 암호화해서 전송한다.
4.WPA2는 WPA의 가장 최근 버전으로 IEEE 802.11i WLAN 보안 표준의 모든 특성 포함한다.
5.SSID는 IEEE 802.11 표준으로 연결하려는 네트워크를 알려주는 식별자로서 Wi-Fi 네트워크의 이름이라고 생각하면 된다.

9강에서는 OSI 참조 모델의 개념과 계층 구조 및 각 계층의 기능에 대하여 학습 하였습니다.
1.OSI 참조 모델은 다음 그림과 같이 각각 특정 기능을 수행하는 계층 7으로 이루어져 있다.

10강에서는 TCP/IP 인터넷 표준 프로토콜 구조, 기능과 주소방식에 대하여 학습하였습니다.
1.TCP/IP 프로토콜은 네트워크 접속 계층, 네트워크 계층, 전송 계층, 응용 계층으로 구성
2.물리 주소(MAC)는 이더넷에서 호스트를 구별하기 위해 모든 이더넷 카드에 개별적으로 부여하는 고유한 48비트의 식별 번호로, 회사에서 제품을 제작할 때 할당 된다.
3.현재 인터넷에 연결된 호스트를 유일하게 식별할 수 있는 32비트 주소 체계를 IP 인터넷 주소로 사용하며, IP 주소를 중복해서 사용할 수 없다.

11강에서는 TCP/IP 프로토콜의 네트워크 접속계층의 구조와 기능에 대하여 학습하였습니다.
1.TCP/IP 계층의 네트워크 접속계층은 물리적 링크 구성과 흐름제어, 오류제어 등을 담당하며, OSI 참조모델의 물리계층과 데이터링크 계층이 이에 해당한다.
2.이더넷 헤더는 수신지 MAC 주소(6바이트), 송신지 MAC 주소(6바이트), 유형(2바이트)의 총 14바이트로 구성되어 있다.

12강에서는 TCP/IP 네트워크 계층의 구조와 기능 및 IP 주소 개념에 대하여 학습하였습니다.
1.TCP/IP 프로토콜에서 네트워크 계층 프로토콜은 IP, IGMP, ICMP, ARP, RARP 로 구성된다.
2.현재 인터넷에 연결된 호스트를 유일하게 식별 할 수 있는 32비트 IP주소를 인터넷 주소로 사용하며, IP 주소를 중복해서 사용할 수 없다.
3.IP 주소 체계는 IP 주소를 효율적으로 배정하려고 클래스라는 개념을 도입했다.
4.클래스에는 A, B, C, D, E 다섯 종류로 실제로 사용하는 것은 A, B, C 클래스 세 종류
5.D 클래스는 IP 멀티 캐스팅 용도, E 클래스는 자원을 확보하려고 예비용으로 분류
6.IP 주소를 보다 더 효율적으로 활용할 수 있도록 네트워크를 분할하는 것을 서브넷팅, 분할된 네트워크를 서브넷이라고 한다.

13강에서는 TCP/IP 네트워크 계층 IP프로토콜 구조와 검사합 및 단편화 과정에 대하여 학습하였습니다.
1.IP는 간단한 오류 검사 기능만 있고, 신뢰성이 보장되지 않는 비연결형 프로토콜이다. 따라서 IP는 패킷을 수신지 까지 최선을 다해서 전송하지만, 전송 완료까지 신뢰성은 보장하지 않는다.
2.IP 계층의 패킷인 데이터그램은 가변 길이이며, 헤더와 데이터부분으로 구성되며, 헤더 크기는 20~60바이트로 패킷을 전달하는데 필요한 모든 정보를 포함한다.
3.검사합(check sum)은 패킷의 전달 중에 발생할 수 있는 오류에 대한 보호를 수행하기 위해 사용되는 오류 검출 방법이다.
4.데이터그램의 크기를 MTU 이하로 작게 나누어 전송해야 하므로 단편화를 수행하는데, IP 데이터그램의 단편화 및 재조립과 관련된 필드는 식별자(Identification), 플래그(Flag), 단편화 옵셋(Fragmentation offset) 이다.

 14강에서는 네트워크 계층의 주소변환 프로토콜인 ARP, RARP와 인터넷그룹 메시지 프로토콜(IGMP)에 대하여 학습하였습니다.
1.송신 측 시스템이 수신 측 시스템으로 패킷을 전송할 때 이 패킷은 물리적인 네트워크를 통과한다. 따라서 패킷이 수신 측 시스템에 도착하려면 수신 측의 논리 주소뿐 만 아니라 물리 주소도 알아야 한다.
2.ARP는 IP 주소를 받아 네트워크 카드의 물리 주소인 MAC 주소로 변환하는 프로토콜이고, Reverse ARP는 물리 주소인 MAC 주소를 IP 주소로 변환하는 역주소 변환 프로토콜이다.
3.IGMP 프로토콜은 인터넷에 연결한 컴퓨터가 멀티캐스트 그룹을 주위의 라우터에 알릴 수 있는 수단을 제공하는 프로토콜이다.
