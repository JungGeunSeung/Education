1강. 정보환경
1.데이터는 현실 세계에서 수집한 사실이나 값을 말하고 정보는 데이터를 처리한 결과로 의사 결정을 지원한다. 이러한 정보는 유용성을 가진다.
2.정보 시스템이 사용할 자료를 처리하는 수단으로 사용되는 시스템을 정보처리 또는 자료처리 시스템이라 하고 일괄 처리, 온라인, 분산처리 시스템으로 구분된다.
3.어느 한 조직의 응용 시스템들이 공용할 수 있도록 통합, 저장된 운영 데이터의 집합을 데이터베이스라 한다.
4.데이터베이스의 특성은 동시 공용, 내용에 의한 참조, 실시간 접근성, 계속적인 변화가 있다.

2강. 데이터베이스 관리 시스템
1.기존의 파일 중심 데이터 처리 시스템에서는 응용 프로그램별로 별도의 파일을 유지하고 파일 내의 데이터 구조가 응용 프로그램에 반영되어 데이터 종속성과 데이터 중복성의 문제가 있다.
2.파일 중심 데이터 처리 시스템의 단점을 극복하고자 등장한 시스템이 데이터베이스 관리 시스템이다.
3.데이터베이스 관리 시스템의 필수 기능으로 정의 기능, 조작 기능, 제어 기능이 있다.
4.데이터베이스 관리 시스템은 계층 및 네트워크 모델로 부터 시작하여 관계 데이터 모델, 객체지향 데이터 모델, 그리고 객체-관계 데이터 모델 순으로 발전하여 왔다.

3강. 데이터베이스 시스템 1
1.데이터베이스 시스템은 데이터베이스, DBMS, 데이터 언어, 사용자 등으로 구성된다.
2.데이터베이스는 통합된 운영 데이터의 모임이고 데이터베이스 관리 시스템은 사용자의 요구에 따라 데이터베이스에 대한 모든 연산을 수행한다.
3.데이터 언어는 데이터베이스를 정의하고 이를 접근하기 위한 언어로서 기능과 사용 목적으로 데이터 정의어, 데이터 조작어, 데이터 제어어로 구분된다.
4.사용자는 유형에 따라 데이터베이스 관리자, 응용 프로그래머, 죄종 사용자가 있다.
5.3단계 데이터베이스는 뷰에 따라 데이터베이스 구조를 3단계(외부, 개념, 내부)로 나누고 각 단계간의 사상을 수행하여 데이터 독립성을 구현한 데이터베이스 구조이다.

4강. 데이터베이스 시스템2 와 데이터 모델
1.데이터베이스 시스템 구조와 데이터베이스에 접근하기 위하여 공통적으로 사용되는 함수들의 모임인 API가 있다.
2.데이터베이스 시스템의 유형으로서는 중앙 집중식 데이터베이스 시스템과 여러 컴퓨터 시스템에서 분산되어 운영되는 분산 데이터베이스 시스템이 있다.
3.데이터 모델이란 현실 세계를 데이터베이스에 표현하는데 사용되는 도구로서 데이터 구조와 연산, 그리고 제약조건으로 구성되어 있다.
4.데이터 모델은 개념적 모델과 논리적 모델로 구분하고 논리적 데이터 모델에는 전형적으로 관계 데이터 모델, 계층 데이터 모델, 네트워크 데이터 모델이 있다.
5.관계 데이터 모델의 릴레이션에서 사용되는 기본적인 용어로서 투플(Tuple), 속성(Attribute), 도메인(Domain) 등이 있다.

5강. 릴레이션과 무결성 제약
1.관계 데이터 모델에서 릴레이션(테이블)은 집합에서의 카티션 프로덕트의 부분집합이고 개념적으로 릴레이션 스키마와 릴레이션 인스턴스로 구성 된다.
2.릴레이션의 특성으로 투플은 유일하고 속성 값은 원자값을 가지며, 투플과 속성은 무순서이다.
3.투플을 식별하는 키는 수퍼키와 후보키, 기본키, 대체키 그리고 외래키가 있다. 이중 수퍼키는 유일성 만을 만족하고 후보키들은 유일성과 최소성 둘 다 만족한다.
4.무결성 제약으로는 개체 무결성 제약과 참조 무결성 제약이 있다.

6강. 관계 데이터 연산 1
1.원하는 데이터를 얻기 위해 필요한 처리 요구를 릴레이션에 수행하는 관계 데이터 연산은 관계 대수와 관계 해석이 있다.
2.관계 대수는 릴레이션을 처리하기 위한 연산의 집합으로서 원하는 데이터와 방법까지 명시하는 절차적 언어이다.
3.관계 대수는 합집합, 교집합, 차집합 등 일반집합 연산자와 실렉트, 프로젝트, 조인 등의 순수 관계 연산자로 구성되어 있다.
4.실렉트 연산자는 릴레이션에서 조건을 만족하는 투플을 검색하고 프로젝트는 릴레이션의 속성들의 값을 검색한다.

7강. 관계 데이터 연산 2
1.조인 연산이란 두 릴레이션으로부터 연관된 투플들을 결합하는 연산으로서 세타 조인, 동등 조인, 자연 조인, 외부 조인, 세미 조인이 있다.
2.세타 조인은 비교 연산자를 이용하여 다양한 조건을 표시하며, 두 릴레이션의 기본키와 외래키 관계에 해당하는 속성들이 세터 조인 조건을 만족하는 투플로 결과 릴레이션이 구성된다.
3.세타 조인 중 동등 조인과 자연 조인이 있다.
4.외부 조인은 상대 릴레이션에 대응되는 투플을 갖지 못하여도 결과 릴레이션의 투플로 남긴다. 왼쪽 외부 조인과 오른쪽 외부 조인, 완전 외부 조인이 있다.
5.디비전은 R릴레이션의 속성이 S릴레이션의 속성 값을 모두 가진 R의 투플에서 S릴레이션이 가진 속성을 제거한 투플을 가지는 릴레이션이다.

8강. 관계 데이터 연산 3
1.관계 대수의 한계로 관계 대수 연산을 확장한 집단 함수, 그룹화, 세미 조인 등의 연산이 있다.
2.집단 함수로 SUM, SVG, COUNT, MAX, MIN 이 있다.
3.그룹화는 속성 값에 의해서 그룹으로 분할되고, 분할된 각 그룹에 대해 질의를 수행한다.
4.관계 연산에서 관계 해석은 원하는 데이터만 명시하고 질의를 수행하는 방법은 명시하지 않는 선언적인 언어로서 투플 관계 해석과 도메인 관계 해석이 있다.
5.투플 관계 해석은 원하는 릴레이션을 투플 해석식으로 표기하고 도메인 관계 해석은 릴레이션을 도메인 해석식으로 표현한다.

9강. SQL Server 설치 및 수행
1.SQL Server는 Microsoft사가 개발한 모바일 장치에서 엔터프라이즈 데이터 시스템까지 다양한 플랫폼에서 운영되는 통합 데이터 및 분석 솔루션이다.
2.여러 제품 에디션이 있으나 데이터 언어인 SQL을 실습하기 위해서 Express 번젼을 설치하는 과정을 학습하였다.
3.설치 단계 중 인증 모드 결정 단계가 있는데 SQL Server 자체 인증인 혼합 모드로 설치를 수행하였으며, 설치 후 외부 프로그램과 접속하기 위하여 TCP/IP 프로토콜을 사용으로 설정하였다.
4.SQL Server에 접속하여 데이터베이스에 대한 환경 설정과 질의를 수행할 수 있는 GUI 환경을 제공하는 SQL Server Management Studio를 설치하여 수행하였다.
5.SSMS를 사용하여 데이터베이스와 사용자, 그리고 스키마를 생성하는 사례를 실습하였다.

10강. SQL 개요와 데이터 정의어
1.SQL(Structured Query Language)은 ANSI에서 데이터 언어의 표준으로 채택된 언어로서 다양한 상용 관계 DBMS에서 사용되고 있다.
2.SQL은 자연어에 가까운 구문을 사용하여 질의를 표현하는 비절차적인 언어이다.
3.SQL은 데이터 정의어와 데이터 제어어 및 데이터 조작어로 구성된다.
4.데이터 정의어의 종류에는 CREATE, ALTER, DROP이 있다.
5.테이블은 CREATE TABLE 구문으로 생성하며 이때 키와 각 속성의 제약 조건을 부여한다. 여기서 키는 기본키(PRIMARY KEY)와 외래키(FOREIGN KEY)를 식별하여 명세한다.
6.DROP TABLE 구문으로 테이블을 삭제하며 열의 추가 및 삭제를 ALTER TABLE 구문으로 수행한다.

11강. select문 1
1.데이터베이스에서 정보를 검색하는 것은 SELECT문 이고 가장 기본적인 구문 형식은 SELECT ~ FROM 이다.
2.조건을 만족하는 정보를 검색할때에는 WHERE절에 조건을 명시한다.
3.한 SQL문에서 테이블의 별칭을 사용할 수 있다.
4.테이블의 모든 속성이나 속성의 일부를 검색할 수 있으며 이때 속성은 ‘,’로 구분한다.
5.문자열의 일부 비교는 LIKE 연산자, 그리고 NULL 값과의 동등 여부는 IS(IS NOT) 연산자를 사용한다.
6.다수의 조건들은 논리 연산자로 결합하여 표시하고 집합 연산자는 IN이다.

12강. select문 2
1.SELECT문으로 검색된 결과를 특정 속성을 기준으로 정렬하여 표시하는 구문은 ORDER BY이다.
2.릴레이션에서 한 속성에 적용되어 단일 값(요약 값)을 반환하는 것은 집단함수이다.
3.GROUP BY는 주어진 속성의 값에 따라 릴레이션(테이블)을 그룹으로 분할한다. 이떄 SELECT문은 분할된 릴레이션을 대상으로 검색을 수행한다.
4.GROUP BY에 의해 분할된 그룹을 대상으로 조건을 적용한다.
5.조인은 여러 테이블의 연관된 투플들을 하나의 테이블로 결합하여 준다.
6.조인에서 속성 이름의 중복을 해결하기 위하여 속성 이름 앞에 테이블 이름을 사용한다.
